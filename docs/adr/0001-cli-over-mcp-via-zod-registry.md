# Replace the MCP server with a CLI built on an in-house zod-driven registry

## Status

accepted

## Context

The MCP frontend loads ~90 tool schemas into every agent's context that uses saarthi — a permanent, large context tax that is the project's main cost. A single `saarthi` CLI invoked through the agent's Bash tool collapses that to one tool plus on-demand `--help`, and also satisfies a secondary goal: shell scripting. We are replacing the MCP frontend outright (not coexisting), in one migration.

## Decision

Build the CLI on a small **in-house registry** that mirrors the existing `server.registerTool(name, { inputSchema, annotations }, handler)` signature, keeping the zod `inputSchema` of each tool as the **single source of truth**. A ~100-line argv→object parser feeds each command's existing zod schema (`schema.parse`) for coercion, defaults, and validation; `--help` is generated by walking the zod shape. Commands follow a `saarthi <noun> <verb>` tree mirroring the handler modules.

We rejected a CLI framework (commander/yargs): it would force re-declaring all ~60 commands' flags in a second DSL, drift from the zod schemas, and add a dependency — turning a near-mechanical handler port into 60 hand translations. Keeping the registry signature identical to `registerTool` makes the ~60 handler bodies near copy-paste (they already return `{ content, structuredContent }`, which the CLI formats as text or, with `--json`, as the raw structured payload), which is what makes the one-shot migration safe.

## Consequences

- Output contract: human text by default, `--json` emits structured payload verbatim, errors/logs to stderr, exit codes mapped from `HyprlandError` codes so scripts can branch.
- Screenshots return a **file path** (agent must `Read` it); a CLI over Bash cannot return inline image content the way MCP did.
- In-memory state the long-lived server held must move to `~/.local/state/saarthi/`: the **grid session**, the **launch rate limiter**, the **session id** (via `SAARTHI_SESSION_ID` env), and the **overlay task** (start/ping/complete).
- New code to own and test: the argv parser and the `--help` generator (zod introspection).
- A **parity contract test** (run while the MCP frontend still exists) asserts the CLI registry exposes the same command set and schemas — the safety rail for the big-bang port.
